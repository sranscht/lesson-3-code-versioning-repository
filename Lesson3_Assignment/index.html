Highlight #1 Versioning Control

Version Control is a repository of files. It is helpful for maintaining documentation and files. Version control can be helpful for designers and developers, helping simplify the process of creating websites. It provides clarity of the contents of the changes and why they were changed.

This is a straight forward process that helps developers not lose information while keeping organized.

Highlight #2 GitHub

GitHub goes above and beyond the basics of version control. Many version control systems require complicated processes. You don't need any complicated servers with GitHub. You can have mulitple people set up the git control. You can create a directory by using init -m 'myproject' to store the history of snapshots. Git add is the command that notices the file and holds them to be committed. Then Committ permentatly stores the files.

It is so simple and effective to use, and makes version control impossible to lose information.

Highlight #3 Remote and Local Repositories

The difference remote and local repositories is the local is usually stored on one's computer as the remote is on the server. Git Init will upload the information onto the local. You can push your local repo to any other repo, which makes Git Great!!

So how the local is it starts at the working directory and then once you enter git add is then staged(shopping cart), then git commit is sent to local repo and finally git push is sent to the remote repo.
